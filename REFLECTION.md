# Reflection

## What Went Well

- Fast MVP iteration
- Clean audit flow
- Shareable reports improved usability

## Challenges

- Designing recommendation logic
- Balancing scope vs polish
- Managing backend integration quickly

## What I Would Improve

- Real AI-generated summaries
- Subscription usage tracking
- Authentication system
- Better analytics dashboard

## AI Assistance Disclosure

AI tools were used during development for:
- debugging
- code suggestions
- architecture guidance
- documentation drafting

All implementation decisions, integration work,
and final code review were completed manually.

## Key Learning

Building a useful SaaS product requires balancing
engineering speed, user experience, and realistic scope.
